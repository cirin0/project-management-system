# AI Agent Guidelines for PM System

This document helps AI coding agents understand and contribute to the PM System project, a **Laravel 13 + Lighthouse GraphQL API** with Pest testing and Laravel Sanctum authentication.

## Project Quick Start

### Build & Run Commands

```bash
# One-time setup
composer run setup

# Development
composer run dev

# Run all tests
composer run test

# Run specific test
php artisan test --compact --filter=TestName

# GraphQL IDE
Open http://localhost:8000/graphiql
```

## Project Structure

```
graphql/              → Lighthouse GraphQL schema definitions
  schema.graphql      → Query, Mutation, Subscription types + User model binding
app/
  Http/Controllers/   → Keep minimal; most logic lives in GraphQL schema/resolvers
  Models/             → Eloquent models with GraphQL type bindings
tests/
  Feature/            → Pest integration tests (HTTP/GraphQL requests)
  Unit/               → Pest unit tests
```

## Core Technologies & Conventions

### 1. GraphQL + Lighthouse Directives

All API endpoints are **GraphQL-first**. No traditional REST controllers. The schema in `graphql/schema.graphql` binds directly to Eloquent models using Lighthouse v6.66+ directives:

- `@find` — Single model lookup by ID or attribute
- `@paginate` — Paginated collections with cursor/offset pagination
- `@eq`, `@where` — Query filtering on fields
- `@rules` — Laravel validation applied to GraphQL arguments
- `@scalar` — Custom scalar types (e.g., DateTime)

**Example Pattern:**
```graphql
type Query {
  user(id: ID @eq, email: String @eq): User @find
  users(name: String @where(operator: "like")): [User!]! @paginate(defaultCount: 10)
}
```

This resolves to: `User::where('id', $id)->first()` or `User::paginate()` without custom resolver code.

Use Laravel Sanctum for API authentication when a query or mutation needs authentication. Prefer `auth:sanctum` and personal access tokens over session-based auth.

### 2. Pest v4 Testing Framework

All tests use **Pest 4** syntax (`test()` / `it()` + `expect()` assertions). See [.agents/skills/pest-testing](./agents/skills/pest-testing/SKILL.md) for complete patterns.

**Key conventions:**
- Feature tests in `tests/Feature/` for HTTP/GraphQL requests
- Unit tests in `tests/Unit/` for isolated logic
- `RefreshDatabase` trait available but commented out in `tests/Pest.php` — uncomment per-test if needed
- Use `->graphQL()` helper to query GraphQL endpoint
- `expect()` over `$this->assert*()`

**Test file creation:**
```bash
php artisan make:test --pest UserQueryTest      # → tests/Feature/UserQueryTest.php
php artisan make:test --pest --unit UserTest    # → tests/Unit/UserTest.php
```

### 3. Laravel Best Practices

See [.agents/skills/laravel-best-practices](./agents/skills/laravel-best-practices/SKILL.md) for comprehensive patterns.

**Quick checklist:**
- ✅ Type hints on all properties and methods (PHP 8+)
- ✅ Define `$fillable` on every Eloquent model
- ✅ Use `with()` to prevent N+1 queries
- ✅ Form Requests for validation (not inline)
- ✅ Query scopes for reusable constraints
- ✅ No raw SQL with user input

---

### 4. Modern PHP 8.3+ Attributes

Models use PHP 8 attributes instead of class properties:

```php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Factories\HasFactory;

#[Fillable(['name', 'email', 'password'])]
#[Hidden(['password', 'remember_token'])]
class User extends Model
{
    use HasFactory;
    
    protected function casts(): array
    {
        return [
            'email_verified_at' => 'datetime',
            'password' => 'hashed',
        ];
    }
}
```

### 5. Database Setup

- **Development**: SQLite file-based (`database/database.sqlite`)
- **Testing**: SQLite in-memory (`:memory:`)
- **Migrations**: Auto-run on `composer run setup`
- Migrations use standard Laravel conventions in `database/migrations/`

---

### 6. API Authentication

- **Method**: Laravel Sanctum (stateless token-based)
- **Route middleware**: `auth:sanctum` on protected GraphQL queries/mutations
- **Token generation**: User has `createToken()` method from Sanctum
- **Keep auth logic** inside GraphQL access control and middleware, not separate REST endpoints

---

### 7. Auto-Generated Files (Do Not Edit)

These files are auto-generated by `php artisan lighthouse:ide-helper`:
- `schema-directives.graphql`
- `programmatic-types.graphql`
- `_lighthouse_ide_helper.php`

Always regenerate them after schema changes.

## Common Tasks

### Add a GraphQL Query

1. **Update schema** in `graphql/schema.graphql`:
   ```graphql
   type Query {
     posts: [Post!]! @paginate
   }
   
   type Post {
     id: ID!
     title: String!
     author: User! @belongsTo
   }
   ```

2. **Create/Update Model** with `#[Fillable]` attribute:
   ```php
   #[Fillable(['title', 'content', 'user_id'])]
   class Post extends Model { ... }
   ```

3. **Regenerate IDE helpers**:
   ```bash
   php artisan lighthouse:ide-helper
   ```

4. **Write tests** in `tests/Feature/`:
   ```php
   test('queries posts', function () {
       Post::factory()->count(5)->create();
       $response = $this->postJson('/graphql', [
           'query' => '{ posts { data { id title } } }'
       ]);
       $response->assertSuccessful();
   });
   ```

### Create a Database Migration

```bash
php artisan make:migration create_posts_table
```

Edit `database/migrations/` and run:
```bash
php artisan migrate
```

### Add a Test

```bash
php artisan make:test --pest MyFeatureTest
```

Then write Pest syntax:
```php
test('verifies something', function () {
    expect(true)->toBeTrue();
});
```

## Project-Specific Patterns

| Pattern | Rationale |
|---------|-----------|
| GraphQL-first, no REST | Single unified API layer, better for modern clients |
| Lighthouse directives | Reduces boilerplate; validation & eager-loading built-in |
| Pest `expect()` syntax | More readable assertions than PHPUnit; modern PHP testing |
| PHP 8 attributes | Type-safe, IDE-friendly, zero runtime overhead |
| Factory injection via generics | IDE autocomplete for factory methods |
| Sanctum auth | Lightweight token-based auth for protected GraphQL operations |
| GraphQL-only app | No REST layer, no Blade UI, keep the surface area small |

## When Something Breaks

| Symptom | Fix |
|---------|-----|
| Schema mismatch warnings | Run `php artisan lighthouse:ide-helper && php artisan config:clear` |
| Tests fail with "table not found" | Run `php artisan migrate:fresh` or enable `RefreshDatabase` trait in test |
| Type hints missing on model properties | Regenerate IDE helpers: `php artisan ide-helper:models --write` |
| Lighthouse resolver not found | Ensure model exists in `app/Models/`, check `#[Fillable]` attribute |
